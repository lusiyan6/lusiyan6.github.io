ASP.NET WebForm

ASP.NET简介​

ASP.NET 是一个使用 HTML、CSS、JavaScript 和服务器脚本创建网页和网站的开发框架。

ASP.NET 支持三种不同的开发模式： Web Pages（Web 页面）、MVC（Model View Controller 模型-视图-控制器）、Web Forms（Web 窗体）

深入了解​

  1. ASP.NET是一种服务器端脚本技术，可以使（嵌入网页中的）脚本由internet服务器执行
    1. ASP.NET是一项微软公司的技术
    2. ASP指的是动态服务器页面
    3. ASP.NET是一种在IIS中运行的程序
  2. IIS简介
    1. IIS是微软的internet服务器
    2. IIS是Web应用程序运行的服务器（用来运行Web程序的工具）
  3. 什么是ASP.NET文件
    1. ASP.NET文件类似于HTML文件
    2. ASP.NET文件可以包含HTML、XML以及脚本
    3. ASP.NET文件中的脚本在服务器上执行
    4. ASP.NET文件的文件名后缀是".aspx"
  4. ASP.NET如何工作
    1. 当浏览器请求HTML文件时，服务器会返回该文件
    2. 当浏览器请求ASP.NET文件时，IIS会把该请求传递给服务器上的ASP.NET引擎
    3. ASP.NET引擎会逐行读取该文件，并执行文件的脚本
    4. 最后，ASP.NET文件会以纯HTML的形式返回浏览器

ASP.NET页的结构​

Page指令​

<%@ Page 
Language="C#" //指明页面代码和后置代码所使用的脚本语言
AutoEventWireup="true" //设置是否自动调用页面事件，默认为true
CodeBehind="WebForm1.aspx.cs" //指定代码后置文件，该文件包含此页面的局部类
Inherits="WebApplication2.demo.WebForm1" //代码后置的类，局部类（项目名.文件名.窗体名）
%>
  • @Page指令定义了ASP.NET页用于编译和解析的属性。每个aspx页面只有一个@Page指令
  • c#代码命名空间用于指定该类所在的位置(项目名.文件名)

Page对象事件​

load事件​

  • 只要页面一刷新，这个方法就会执行
  • 要想让这个方法里某些逻辑不重复执行，只需要利用Page对象的属性，isPostBack是否回发
  • Page是全局变量，在使用时无需Page.isPostBackd
  • 第一次页面加载，isPostBack=false，其他为true
  • 不管是第一次加载还是回发，load事件都会执行

回发机制​

  • 非第一次加载会产生回发机制
  • 产生时机：当用户操作让表单产生submit产生回发
  • 原理分析：当前页面被提交给服务器处理，处理时服务器会对页面进行重绘
  • 网页拖拽一个按钮控件过来，默认类型是submit
